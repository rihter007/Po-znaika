Протокол взаимодействия клиента с серверной стороной:

Все действия идут поверх протокола HTTPS
Условный URL обращения помечен как [URL] (например software.po-znaika.ru)

Внутренние коды ошибок сервера:
crc32(po-znaika.internal_status.eAlreadyExists)   // данное действие записи уже выполнено
сrc32(po-znaika.internal_status.eNotFound)        // нет такого параметра (например упражнения)
crc32(po-znaika.internal_status.eInvalidArgument) // некорректный параметр

1. Запрос лицензирования
Url: [URL]\license

Клиент:
{Headers:}
Login: login
Password: password

Сервер:
200 OK
{Headers:}
License: Commercial - текущая коммерческая лицензия активна
         Expired - была коммерческая лицензия, но на текущий момент закончилась

401 - некорректный логин/пароль

2. Выставление оценки за упражнение
Url: [URL]\exercise_score

Клиент:
{Headers:}
Login: login
Password: password
Exercise_Id: [id, see above]
Score: [score]
Date: [Http date format] (Example: Wed, 15 Nov 1995 06:25:24 GMT)

For future usage: Action_Unique_Id: [SHA1(Логин.Решение.Программа.Язык.имя_упражнения.дата.оценка] // уникальный идентификатор действия, чтобы не дай бог одно и тоже не послать 100 раз

Сервер:

200 OK - запись добавлена (возможно такая уже была)
401 - некорректный логин/пароль
400 - некорректные данные (See internal status header)
Headers: (варианты)
Internal_Status: code (см. коды ошибок сервера)

3. Получение результатов выполнения упражнения

Url: [URL]\diary

Клиент:
{Headers:}
Login: login
Password: password
Exercise: Внутреннее имя упражнения или группы упражнений (по принципу совпадения exercise_name* - т.е. любые упражнения, входящие в данную группу)
Start_Date: [Optional] Дата, начиная с которой требуются оценки
End_Date: [Optional] Дата, заканчивая которой требуется получить оценки

Сервер:

401 - некорректный токен аунетнтификации
400 - некорректные данные (See internal status header)
Internal_Status: code (см. коды ошибок сервера)

200 OK
